| 20:35 - UI vs API |
+-------------------+

둘 다 Interface라는 단어를 내포하고 있다

사람과 기계가 소통하기 위한 접점

스마트폰의 전원과 디스플레이 -> 사람과 소통하기 위한 Interface

USB Interface

웹 애플리케이션 <-> 웹브라우저

prompt("비밀번호를 입력해주세요")

사용자가 웹 화면에서 사용하는 UI

JavaScript의 문법에 따라 프로그래밍 적으로 시스템에게 명령하는 것 API

prompt("비밀번호를 입력해주세요") <- 우리가 만든것이기도 하면서 아니기도 함
------  -----------------------

웹브라우저가 미리 만들어 놓고 준비하고 있는 것을, 우리가 prompt("")라는 함수를 실행(API를 통해서 가져다 씀)함으로써 웹브라우저에게 명령하는 것

API - 애플리케이션 프로그래밍 적으로 요청하는 Interface

UI <-> 웹애플리케이션 <-> 웹브라우저 <-> OS <-> C <-> 어셈블리어 <-> 기계어 <-> 전자공학(CPU, RAM 등의 HW) <-> 전기공학 <-> 물리학

각 계층은 API를 통해서 교류하고 있다.

API 전성시대

과거에는 API를 제공하는 시스템이 OS 정도가 있었다

웹브라우저.. OS 입장에서는 수 많은 소프트웨어 중 하나

그러나 그 동시에 스스로가 API를 갖음

웹브라우저 위에서 동작하는 수많은 웹 애플리케이션 등장하게 됨

수 많은 웹 애플리케이션 중에 사람들이 많이 사용하는 몇 가지 웹 애플리케이션(구글, 페이스북, 트위터)이 등장

이런 웹 애플리케이션 또한 스스로가 API를 갖음

!!?? API의 폭발적 증가

스마트폰 -> API 제공 -> 여러 애플리케이션!! 폭발적 증가
