웹 앱 만드는 순서

1. 구상

2. 기획(계획): 구상한 것을 좀 더 세부적으로 정의함

3. 디자인: 웹사이트를 아름답게 보이게 하는것, 사용자가 편히 사용할 수 있도록 사용자 경험을 향상시키는 것

4. 개발(Development, programming, engineering): 기획한 것을 동작하게 하는 것

5. 테스트(test, QA;Quality Assurance)

위의 순서가 꼭 엄격하게 지켜지는 것은 아님

또한 위의 순서를 계속 반복하면서, 현실의 필요를 충족시켜가는 과정을 계속해서 꾸준히 해나가야함

이런 반복적인 작업을 잘하기 위해서 중요한 개념이 있다.

서비스도 이것과 비슷하다고 생각함.

처음 서비스를 만드는 사람이 이 서비스는 어떠해야 한다는 주관이 있을 것.

여러 사람이 이 서비스를 이용하면서 발생하는 어떤 문화 등을 잘 뒷받침 해주는 것이 좋은 접근 방법.

처음부터 모든 기능을 다 만드는 것 보다.

작게 시작해서, 필요하다고 생각되는 기능을 신중하게 추가하는게 좋다고 생각.
